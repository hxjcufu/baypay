# BayPay — сборка в .ipa

Это готовый Capacitor-проект: он оборачивает HTML-прототип BayPay в реальное
iOS-приложение (WebView-контейнер). Дизайн, анимации и вся логика — те же,
что были в браузерном прототипе, просто теперь это нативное приложение.

## Важно — то, что нельзя обойти

Собрать `.ipa` можно только там, где есть **Xcode + macOS** (правило Apple,
не моё ограничение). У меня в чате Linux-контейнер без Xcode, поэтому файлы
ниже нужно собрать одним из двух способов — оба реальны и не требуют
подписывать приложение платным аккаунтом разработчика.

Также: чтобы поставить `.ipa` на **настоящий iPhone** (не симулятор), Apple
всё равно требует минимальную подпись — но это можно сделать бесплатно
через обычный Apple ID (см. способ 2).

---

## Способ 1 — облачная сборка без Mac (Codemagic)

1. Зарегистрируйся на https://codemagic.io (бесплатный тариф даёт часы сборки в месяц)
2. Залей эту папку в свой репозиторий на GitHub/GitLab
3. Подключи репозиторий в Codemagic — он сам подхватит `codemagic.yaml` из этой папки
4. Запусти workflow **baypay-ios-unsigned** — на выходе получишь `.app` для
   симулятора (можно открыть в Xcode Simulator на любом Mac или в облачном
   Mac-эмуляторе)
5. Если нужен `.ipa` на реальный телефон — запусти **baypay-ios-signed**,
   но для этого в настройках Codemagic нужно будет привязать Apple ID
   (бесплатный аккаунт подходит для ad-hoc подписи на 7 дней)

## Способ 2 — сборка на своём/чужом Mac (5 минут)

```bash
npm install
npx cap add ios
npx cap sync ios
npx cap open ios
```

Откроется Xcode. Дальше:
1. Подключи iPhone по кабелю или выбери симулятор
2. Xcode → Signing & Capabilities → выбери свой Apple ID (бесплатный, без
   подписки разработчика) как Team
3. Нажми ▶️ Run — приложение установится и запустится
4. Чтобы получить файл `.ipa`: Product → Archive → Distribute App →
   Development → Export

## Структура проекта

```
baypay-capacitor/
├── www/index.html       ← сам прототип (интерфейс приложения)
├── capacitor.config.json
├── package.json
└── codemagic.yaml       ← готовый конфиг для облачной сборки
```

Если поменяешь дизайн — правь `www/index.html` и заново прогони
`npx cap sync ios`.
